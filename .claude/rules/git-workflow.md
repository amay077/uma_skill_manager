# Git 操作・コミット規約

## 操作のガードレール

- **禁止操作**（ユーザー指示がある場合のみ）：`git push`, `git reset --hard`, `git stash drop/pop/branch`, `git branch -D`, `rm -rf`
- **要承認操作**（ユーザー指示または作業フロー内で許可時のみ）：`git commit`, `git checkout`, `git merge`, `git stash apply`
- **安全操作**（自由に実行可能）：`git status/diff/log/show`, `cat/ls/grep/find`, Read/Grep/Glob ツール

## Git コミット後の振り返り

git コミット後、以下を実施：
1. 学び・教訓の列挙（エージェントが知らなかったことを含む）
2. ドキュメント追記・修正の提案（後学者の理解を助けるため）

## コミットメッセージフォーマット

```
{PROJECT_KEY}-{TASK_ID} 変更内容の概要

- 詳細な変更点

🤖 Generated with {AGENT_NAME}({AGENT_MODEL}) - {AGENT_URL}
```

※ PROJECT_KEY, TASK_ID は README.md 参照。不明時は指示者に問い合わせ。
※フッターは1行のみ（`Co-Authored-By` 等を含めない）。

## 課題管理システムコメント署名

課題管理システムへのコメントには、以下の署名を末尾に追加（空行1つ開ける）：

```
🤖 Generated by {AGENT_NAME}({AGENT_MODEL}) - {AGENT_URL}
```
