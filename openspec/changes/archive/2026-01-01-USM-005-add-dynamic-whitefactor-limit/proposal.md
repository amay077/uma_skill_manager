# 白因子下限の動的調整機能

## Why

ウマ娘 DB のスキル検索では、サイト側の制限により1スキルあたり最大100件しか表示されない。このため、検索結果が100件を超えるスキルでは、101件目以降に存在する「最適なフレンド」を見逃す可能性がある。

白因子合計数下限を動的に調整することで、各スキルの検索結果を100件以下に収め、全ての候補を取得可能にする。

## What Changes

1. **検索ロジックの変更**: スキル検索時、結果が100件を超える場合は白因子下限を5ずつ増加させて再検索を繰り返し、100件以下になったら結果を取得する
   - 増加値「5」は現在固定値（将来的に調整可能な設定にする可能性あり）

2. **型定義の拡張**: `SkillSearchResult` に実際に使用した白因子下限値を記録するフィールドを追加
   - CLI で指定された初期値、または動的調整後の値を記録

3. **出力形式の拡張**: Markdown 出力に「スキル別 白因子合計数下限」テーブルを追加し、各スキルで使用した条件を可視化する
   - 実際に使用した白因子下限値と、調整後の最終結果件数を表示

## Impact

- **Affected specs**: `uma-db-scraping`
- **Affected code**:
  - `scraping/src/uma-db/types.ts` - 型定義追加
  - `scraping/src/uma-db/client.ts` - 検索ロジック変更
  - `scraping/src/utils/output.ts` - 出力形式変更
- **Breaking changes**: なし（後方互換性あり）
