# レスポンシブ対応の改善

## Why

現在のレスポンシブ対応は 767px のブレークポイントのみで、タブレット（768px - 1024px）向けの最適化がない。また、spec で定義されている「詳細検索パネルの折りたたみ機能」が未実装であり、モバイルでの UI 体験に改善の余地がある。

## What Changes

- **タブレット対応追加**: 768px - 1024px のブレークポイントを追加し、間隔・フォントサイズを調整
- **モバイル対応強化**:
  - 順位チェックボックス: 5 列 → 3 列（視認性向上）
  - 作戦・距離等のチェックボックス: 縦並び → 2 列グリッド（スペース効率化）
  - 詳細検索パネル: 折りたたみ可能に（spec 準拠）
  - スキルカード: パディング縮小
- **ブレークポイント体系の整理**: Desktop（>= 1025px）、Tablet（768px - 1024px）、Mobile（<= 767px）

### 既存 spec シナリオからの変更点

既存の `Responsive Design` 要件には 2 つのシナリオがありましたが、これを 6 つのシナリオに拡張・詳細化します：

**既存シナリオ「Mobile layout」の拡張**:
- 元の記述: 「画面幅に応じたレイアウトで表示される」「検索フォームが縦並びで表示される」「スキルカードが 1 列で表示される」
- 新規シナリオ:
  - **Desktop layout**: デスクトップ（>= 1025px）での表示を明示化（検索フォームが横並び、詳細パネル常時展開）
  - **Tablet layout**: タブレット（768px - 1024px）向けの新規最適化（中間サイズの間隔・フォント）
  - **Mobile layout**: モバイル（<= 767px）の詳細を具体化（順位チェックボックス 3 列グリッド、作戦・距離等 2 列グリッド、詳細パネル初期折りたたみ）

**既存シナリオ「Filter panel toggle」の分割**:
- 元の記述: 「詳細検索パネルは初期状態で折りたたまれている」「タップで展開・折りたたみが可能である」
- 新規シナリオ:
  - **Filter panel initial state**: 初期折りたたみ状態の明示化
  - **Filter panel expand**: 展開操作の詳細（ボタンクリック、アイコン変化、パネル表示）
  - **Filter panel collapse**: 折りたたみ操作の詳細（ボタンクリック、アイコン変化、パネル非表示）

この変更により、既存の簡潔な記述を実装に必要な詳細レベルまで拡張し、テスト可能なシナリオとしています。

## Impact

- **Affected specs**: `skill-search-frontend`（Responsive Design 要件の MODIFIED）
- **Affected code**:
  - `web/public/css/style.css` - メディアクエリ追加・強化
  - `web/public/index.html` - 折りたたみトグルボタン追加
  - `web/public/js/app.js` - 折りたたみロジック追加
- **Breaking changes**: なし（既存の表示は維持しつつ改善）
