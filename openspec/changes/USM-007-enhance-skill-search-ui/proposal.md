# スキル検索 UI の機能強化

## Why

現在の検索 UI には以下の課題がある：

1. **種別フィルタがラジオボタン**: 複数種別の同時選択ができない（固有と進化を両方検索したい場合など）
2. **詳細検索の折りたたみ**: 毎回展開が必要で操作が煩雑
3. **順位条件が不十分**: 1〜9位の個別指定ができない（例：3位のみ）
4. **効果量・効果時間が非表示**: 検索結果からスキルの強さを比較できない
5. **ソート順が固定**: 効果量でのソートができない
6. **ビットフラグ検索の不統一**: チェックボックスの ON/OFF の解釈が不明確

## What Changes

### UI 変更

- 種別フィルタをラジオボタンからチェックボックスに変更（進化/固有/継承固有/白）
  - 白スキル: type = 'normal' のスキル
  - 継承固有: sub_type = 'inherited_unique' のスキル
- 詳細検索パネルの折りたたみを廃止し、常時表示に変更
- 順位条件を 1〜9位の個別チェックボックスに変更
- 効果種別フィルタをラジオボタンからチェックボックスに変更

### 検索ロジック変更

- ビットフラグ系フィルタの統一ロジック（作戦、距離、バ場、フェーズ、順位）:
  - 全チェック ON = 全チェック OFF → ビットフラグが全て 1 のスキルを検索（条件限定なし）
  - 一部チェック → チェックされたビット位置が 1 のスキルを OR 検索
- 種別フィルタ:
  - 全チェック ON = 全チェック OFF → 全種別を検索
  - 一部チェック → OR 検索
- 効果種別フィルタ:
  - 全チェック ON = 全チェック OFF → 全効果種別を検索
  - 一部チェック → OR 検索

### 結果表示変更

- スキルカードに効果量と効果時間を表示
- ソート順を効果量×効果時間の降順に変更
- 効果種別が複数チェック時は左端の効果種別で効果量を算出

## Impact

- **Affected specs**: skill-search-frontend（既存仕様の修正）
- **Affected code**:
  - `web/public/index.html`（フォーム UI）
  - `web/public/js/db/queries.js`（検索ロジック）
  - `web/public/js/db/constants.js`（定数追加）
  - `web/public/js/components/SearchForm.js`（フォーム値取得）
  - `web/public/js/components/FilterPanel.js`（フィルタ状態管理）
  - `web/public/js/components/SkillCard.js`（効果量・効果時間表示）
  - `web/public/js/app.js`（検索オプション組み立て）
  - `web/public/css/style.css`（レイアウト調整）
- **Breaking changes**: なし（既存の検索結果に影響するがデータ破壊なし）
